1 CosmicFish: a cosmology forecasting tool
2 ========================================
3 
4 This library contains a set of tools and applications to
5 perform forecast of the accuracy of future cosmological
6 experiments.
7 
8 If you find this collection useful, feel free to download, use it and suggest
9 pull requests!
10 
11 The official distribution is available for download at:
12 
13 http://cosmicfish.github.io
14 
15 while the official developers version is available on GitHub, and you can clone the
16 repository using:
17 
18  git clone https://github.com/CosmicFish/CosmicFish
19 
20 ### 1. CosmicFish Requirements:
21 
22 Currently, the distributed source code requires the intel fortran compiler and some other stuff.
23 Here's a list of dependencies for a full build:
24 
25  ifort or gfortran (5.3.0)
26  MKL or BLAS/LAPACK
27  numdiff
28  doxygen
29 
30 If you use ifort the code will automatically try to link mkl while if you use gfortran you need BLAS and LAPACK libraries installed. You can do so by ``sudo apt-get install liblapack-dev``. If both ifort and gfortran are installed the code will use ifort. Compilation options can be found in ``make_macros/compilers_options.make``.
31 
32 Notice that only the 5.3.0 version of gfortran is supported as previous versions missed some OOP features. You should (hopefully) be able to get this version of the code by:
33 
34  sudo add-apt-repository ppa:ubuntu-toolchain-r/test
35  sudo apt-get update
36  sudo apt-get install gcc-5 g++-5 gfortran-5
37 
38 The python library then requires several other libraries to work. We warmly suggest to install a
39 bundled package like (https://www.continuum.io/downloads). Just in case here's a list of all python
40 dependencies:
41 
42  python 2.7
43  python library matplotlib
44  python library os
45  python library scipy
46  python library numpy
47  python library math
48  python library itertools
49  python library copy
50 
51  nosetests for python testing
52  with coverage (pip install coverage should work)
53  sphinx (http://www.sphinx-doc.org) for python documentation
54 
55 Some of the requirements of the library are included and will be compiled along with the library if not present on the system.
56 These can de found in the folder:
57 
58  cosmicfish/bundled

60 ### 2. Installation procedure:
61 
62 After downloading the CosmicFish library it can be compiled by running:
63 
64  make cosmicfish
65 
66 and tested using:
67 
68  make run_test
69 
70 If you want to build the CosmicFish library in parallel, using multiple processors, just add ``-jN`` to the make commands. If you use ``-j`` the Makefile will use all available processors.
71 
72 ### 3. Examples and packages
73 
74 After the library is built it can be directly used to produce some examples with:
75 
76  make examples
77 
78 This will produce a set of example results in the ``examples`` folder. These are meant to outline a standard workflow with the CosmicFish library.
79 
80 The structure of the example folder can be used to produce CosmicFish packages that naturally exploit the set up workflow.
81 To do so just copy (and rename) the example folder somewhere on your system then tell the system where the CosmicFish library is built with:
82 
83  export COSMICFISH_DIR=path_to_cosmicfish_build
84 
85 Then replace the parameters and the analysis parameters and run everything with ``make all`` in your directory.
86 
87 For further details please check the examples directory readme.

118 ### 7. Build system target
119 
120 The Makefile has several targets. Here's a breakdown of the main ones:
121 
122 **Main**:
123 
124 * ``all``: builds all the library, runs all the tests and builds the documentation;
125 * ``cosmicfish``: builds all the library and the apps, builds the test without running them and builds the documentation;
126 * ``default``: the default target is cosmicfish;
127 * ``clean``: removes all built material leaving the structure intact;
128 * ``distclean``: removes everything;
129 
130 **Secondary**:
131 
132 * ``apps``: compiles the CosmicFish apps;
133 * ``bsolvers``: compiles all the Boltzmann solvers;
134 * ``bundled``: compiles all bundled software;
135 * ``directories``: creates the directory structure of the CosmicFish library;
136 * ``documentation``: builds both the doxygen and the python documentation;
137 * ``test``: builds the CosmicFish tests;
138 * ``run_test``: runs both the CosmicFish tests and Python tests;
139 
140 **Some options**:
141 
142 Compile with debug options:
143 
144  make target DEBUG=1
145 
146 Change the build directory:
147 
148  make target BUILD_DIR=wherever_you_want_to_build
